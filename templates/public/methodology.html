Methodology

How hosts are ranked

Manual tiers

Recommendations are editorial. The CMS stores tier, drag-saved order, category picks, tags, pros, cons, and caveats directly.

CPU context

CPU model, vendor, core allocation, base GHz, peak GHz, memory limits, memory speed, storage, locations, and panel data are shown when verified or published by the host.

Plans and regions

Plan rows and hardware tiers are stored separately. Directory sorting uses the matching plan/hardware offer so max RAM, CPU cores, region filters, and tier-specific hosting features do not show a misleading host-level aggregate.

Trust notes

Review signals, source URLs, status labels, and caveats are kept separate from performance notes so strong hardware claims do not hide risk.

Static publishing

The public site is generated from local JSON data and served as static files on GitHub Pages.
