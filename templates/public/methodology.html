Methodology

How hosts are ranked

Manual tiers

Recommendations are editorial. The CMS stores hosting type selectors, tier, drag-saved order, category picks, tags, pros, cons, and caveats directly.

CPU context

CPU model, vendor, core allocation, base GHz, peak GHz, memory limits, memory speed, storage, locations, and panel data are shown when verified or published by the host.

Plans and regions

Exact plan rows are used when providers publish clear plan tables. For custom or broad pricing, estimated rows are generated from package-level $/GB pricing, RAM ranges, and CPU options, and are labelled as estimates.

Trust notes

Review signals, source URLs, status labels, and caveats are kept separate from performance notes so strong hardware claims do not hide risk.

Static publishing

The public site is generated from local JSON data and served as static files on GitHub Pages.
